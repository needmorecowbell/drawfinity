# Phase 06: Shape Tools

Add geometric shape tools (rectangle, ellipse, polygon, star) as first-class canvas objects stored alongside strokes in the Yjs CRDT document. Shapes are parameterized geometry rendered via the same WebGL2 pipeline.

## Design Decisions

**Shapes in the same Y.Array as strokes.** Add a `type` discriminator field to each Y.Map entry (`"stroke"` or `"shape"`). Existing items without `type` are treated as strokes for backward compatibility. This keeps undo/redo working through the existing UndoManager without a second scope.

**Parameterized geometry, not point arrays.** Shapes store center, width, height, rotation, and type-specific params (sides, inner radius). Vertex data is generated at render time and cached. This enables precise hit-testing and future resize/rotate handles.

**Rendering through the existing pipeline.** Shape vertex data (triangle strips) is generated by a `ShapeMesh` module analogous to `StrokeMesh`. Shapes participate in spatial indexing and vertex caching like strokes.

## Tasks

- [x] Define shape data model:
  - Create `src/model/Shape.ts` with `Shape` interface: `id`, `type` (`"rectangle" | "ellipse" | "polygon" | "star"`), `x`, `y` (center), `width`, `height`, `rotation`, `strokeColor`, `strokeWidth`, `fillColor` (nullable), `opacity`, `sides?`, `starInnerRadius?`, `timestamp`
  - Create `CanvasItem` discriminated union type: `type CanvasItem = (Stroke & { kind: "stroke" }) | (Shape & { kind: "shape" })` — or use a `kind` field on each
  - Add `generateShapeId()` function
  - Update `src/model/index.ts` barrel exports

- [x] Shape-to-vertex generation:
  - Create `src/renderer/ShapeMesh.ts` with triangle strip generation functions:
    - `generateRectangleVertices(shape)` — 4-corner outline as triangle strip with stroke width
    - `generateEllipseVertices(shape, segments)` — N-gon approximation (32-64 segments)
    - `generatePolygonVertices(shape)` — regular polygon from `sides` param
    - `generateStarVertices(shape)` — alternating inner/outer radius vertices
    - `generateShapeVertices(shape)` — dispatcher by shape type
  - For filled shapes, generate triangle fan vertex data (separate draw mode or degenerate strip)
  - Apply rotation transform to all vertices
  - Create `src/renderer/__tests__/ShapeMesh.test.ts` with tests for each shape type: vertex count, winding order, bounding correctness

- [x] Update CRDT layer for shapes:
  - Add `shapeToYMap()` and `yMapToShape()` to `src/crdt/StrokeAdapter.ts` (or create `ShapeAdapter.ts`)
  - Add `type` field to Y.Map for new items; absent `type` → stroke for backward compat
  - Add to `DrawfinityDoc.ts`: `addShape(shape)`, `removeShape(shapeId)`, `getShapes()`, `getAllItems(): CanvasItem[]`
  - Ensure `getStrokes()` still works (filters by type) so existing code doesn't break
  - Verify undo/redo works for shape add/remove (same Y.Array, same UndoManager)

- [x] Shape capture (input handling):
  - Create `src/input/ShapeCapture.ts`:
    - `pointerdown`: record start point in world coordinates
    - `pointermove`: compute shape preview dimensions from drag extent, render preview overlay
    - `pointerup`: finalize shape into DrawfinityDoc via `addShape()`
    - Hold `Shift` for constrained proportions (square from rectangle, circle from ellipse)
    - Hold `Alt` for center-out drawing mode (start point = center, not corner)
  - Coordinate with `StrokeCapture.ts` — active tool determines which capture handles input
  - *(Completed: ShapeCapture with enable/disable gating, ShapeToolConfig, preview support, 18 unit tests passing)*

  - Update `Renderer.ts` with `drawShape()` or integrate into `drawStrokeBatch()`

- [ ] Update toolbar UI:
